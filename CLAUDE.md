## 项目目标与愿景（Goal & Vision）

南京六部（Nanjing Sixth Department）**考勤自动化项目**。目标是沉淀一套**可复用、可自动化的考勤工作流**，
分三层递进：

1. **近期** —— 完成本期（2026-05）考勤结果文档。
2. **中期** —— 把规则与流程固化成一个 **Skill**，让大模型能完整、可重复地执行整套考勤工作流。
3. **远期（最期望）** —— 做一个 **GUI 上位机**：可视化驱动工作流，自动完成能自动的部分，对需要人工判断的
   数据 / 选择**弹出交互**，最终结合用户输入**协作生成完整考勤文档**，实现考勤全自动 / 半自动。

**核心原则：** 面向「可复用工作流」而非「一次性脚本」。下文「考勤规则」是与用户逐条确认的**权威口径**；
其中「通用规则」要做成与月份无关，「本期落地（2026-05）」是本月的具体数据实例。

## 考勤规则（Authoritative）

### 阶段一 · 表格整理 → 《南京六部（初表）》

1. 删除原表中**部门 ≠ 南京六部**的人员（如「总部综合部」）。
2. 按**姓名**从《员工花名册》匹配 **工号、职位** 填入；再按**工号升序**排序。
   ⚠️ 排序时**整行（含 G–AK 所有打卡时间）必须随姓名一起移动，不能错位**。
3. **重名**：参考花名册 `用工类型`，取**在职**（转正/试用）那条的信息（入离职区间覆盖当月）。
4. **离职人员打卡 < 7 天**：把这类人姓名整理出来，**询问用户**保留哪些；未保留的从表中删除数据。
   被保留的离职人员**只计算到离职日**。
5. 整理后另存为 **《南京六部（初表）》**。

### 阶段二 · 考勤核算（在《初表》上完成 → 《南京六部（考勤）》）

#### A. 工作日 / 节假日判定

- 基准日历看**《调班表》左半区网格**（`N班`=上班 / `N休`=休息）。
- **个人调班（右半区名单）**：每人「原日期/班次」与「现调日期/班次」两列涉及的**当月日期，一律按基准日历"对调/翻转"班↔休**
  ——两列谁=上班/休息**不固定**，由基准决定（基准班→改休、基准休→改班）。例(4月)：某人现调列含 4/1、4/2(基准班)→改休；原列含 4/25(基准休)→改班。
- ⚠️ 月份以**原表 A1 周期**为准；原表与《调班表》**月份必须一致**（不一致应报错，不可硬算）。
- 若未提供《调班表》→ **询问用户**是否补充；不补充则**视为无调班，按正常法定日历**。

#### B. 加班（OT）

- **排除人员**：陆钊颖、曹兆阳 **不计加班**。
- **有效性**：当天 **≥ 2 次打卡**才计 OT。
- **取整**：时间点按 **0.5 小时向内取整**（起点向上、终点向下）。例 `9:15–10:15` → 按 `9:30–10:00` = 0.5h。
- **下限**：**9:00 之前的打卡不计加班**（OT 起点不早于 9:00）。
- **工作日 OT** = 末次打卡(向下取整) − `18:30`；**若 ≥ 2 小时（含）则减 0.5h**；结果 < 0 记 0。
- **节假日 OT** = 末次打卡(向下取整) − 首次打卡(向上取整，且 ≥9:00)；**若区间跨 `12:00–13:30` 则扣除该 1.5h**；
  **若 > 8 小时（不含）则减 0.5h**。
- **外勤**：**工作日外勤不计加班**；**节假日外勤正常计加班，但超过 `21:00` 的时间不计**（末点封顶 21:00）。
- **写法**：在每人打卡行**下方新增一行**，把当天 OT 小时数填到对应日期单元格——**只填数字、无单位**，
  当天无 OT 则**留空**。

#### C. 迟到 / 早退（仅工作日；全员参与，含外勤，无一例外）

- **迟到**：工作日**首次打卡晚于 9:00** → 当日整格**字体标红 `FF0000`**（当天所有时间点都红，不只迟到那次）。
- **迟到加重**：**首次打卡晚于 9:30** → 字体不变，**单元格底色填红 `FF0000`**。
- **刘莹特例**：刘莹**晚于 9:05** 即按上一条处理（底色红）。
- **曹兆阳特例**：迟到**一律仅字体红**（首卡>9:00 即红字），**不适用 >9:30 底色红**（早退等不受影响）。
- **新入职员工·入职当天特例**：入职当天的迟到同样**仅字体红、不底色红**（仅当天；之后按常规）。
- **早退**：**末次打卡早于 18:30** → 字体不变，**底色填红 `FF0000`**。**早退不适用于外勤打卡**，其余全员适用。

#### D. 缺卡 / 公出 / 未出勤

- **一天 2 次打卡 = 有效出勤**（工作日、节假日加班同此口径）。
- **工作日打卡 < 2 次** → **询问用户**归类：
  - 缺卡 → 底色**绿 `92D050`**；公出 → 底色**浅蓝 `DDEBF7`**；未出勤 → 底色**红 `FF0000`**。
- **节假日加班 < 2 次（节假日缺卡）** → **不做任何标记，当日不计 OT**。
- 缺卡与迟到/早退**可并存**；底色冲突时 → **字体红 `FF0000` + 底色绿 `92D050`**。

#### E. 外勤

- **外勤日认定**：当天**只要出现任一「外勤」打卡**，整天即按外勤日处理（混合打卡也算）。
- 外勤日的工作日/节假日判定**与常人相同**（法定日历 + 《调班表》）。
- **外勤日整格底色浅蓝 `DDEBF7`**；该日若打卡 < 2 次出现缺卡/未出勤/公出，按 D 条常规规则处理（覆盖外勤蓝）。

#### F. 背景色优先级与冲突（高 → 低）

1. **工作日 <2 次的归类**（三选一，最高）：未出勤红 `FF0000` / 缺卡绿 `92D050` / 公出蓝 `DDEBF7`。
2. **外勤日**：浅蓝 `DDEBF7`。
3. **迟到 > 9:30 / 早退**：红 `FF0000`（最低）——**外勤日时被外勤蓝覆盖、不生效**。

- **缺卡 + 迟到/早退** 冲突 → **字体红 `FF0000` + 底色绿 `92D050`**。
- **迟到 9:00–9:30** 为**字体红**（不占底色），可与任何底色并存。
- 其余未列冲突 / 特殊 → 询问用户。

#### G. 在职区间

- 仅对员工**在职区间内**（入职日 ≤ 当日 ≤ 离职日；无离职日则到月末）的日期做判定与着色。
- **入职前 / 离职后的工作日空格**：**留空、不标记**（不计迟到/缺卡/未出勤），**也不影响全勤判定**。

#### H. 全勤

- 员工**在职区间内的工作日**无 迟到 / 早退 / 缺卡 / 未出勤 → **全勤**，将其**姓名单元格底色标浅紫 `D09ECE`**。

## 已知坑（PowerShell，已多次踩到）

- **变量名大小写不敏感**：`$DEPT` 与 `$dept` 是同一变量 → 用不同名（非仅大小写）。见 [[powershell-var-case-insensitive]]。
- **函数名勿用内置别名**（`cp`=Copy-Item 会压过 `function CP`）。
- **.ps1 纯 ASCII**（PS5.1 按 GBK 读无 BOM 文件），中文用 `[char]0x....`。见 [[powershell-ps1-encoding-ascii]]。
- Excel 序列号日期 → `[DateTime]::FromOADate(n)`。

## 环境与工具注意事项（Windows，已验证）

- **没有可用的 Python**（Store stub，无 `pip`），**`openpyxl`/`pandas` 不可用**——不要写依赖它们的脚本。
- **读 xlsx**：当 ZIP 解压解析 `xl/worksheets/sheet1.xml`（+ `xl/sharedStrings.xml`）。两种文本格式：原表为
  **inlineStr**（文本在 `<is><t>`），花名册/调班表为**共享字符串**（`t="s"`，`<v>` 为 0 基下标）。别依赖 WPS 的占位 dimension。
- **写 / 生成 xlsx**：用 **`xlsx` skill**（`anthropic-skills:xlsx`）。
- **PowerShell `.ps1` 编码**：PS 5.1 把无 BOM 的 `.ps1` 当 GBK/ANSI 读 → **脚本保持纯 ASCII**，中文用 code points
  构造（如 `([char]0x65B0,[char]0x5B8B,[char]0x4F53) -join ''`=新宋体）。
- **日期换算**：Excel 序列号 → `[DateTime]::FromOADate(n)`。
- **Shell**：Bash 工具是 Git Bash；`cd /d "..."` 会报错，用普通 `cd` 或绝对路径；也可用 PowerShell 工具。
- 内容全程中文，保留 `外勤`/`离职` 等术语原文。`.claude/settings.local.json` 预批准了若干只读命令。
