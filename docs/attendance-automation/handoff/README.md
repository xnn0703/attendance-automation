# 考勤自动化 · 设计交付包

南京六部考勤自动化桌面应用（Windows / PySide6）的完整设计交付。

## 目录

- **设计交付说明.html** — 主文档。色彩 / 字体 / 组件 / 动效 / 交互逻辑 / 数据计算规则的完整规范，所有数值可直接落到代码。**先看这个。**
- **spec.css** — 说明文档的样式（随 html 一起打开即可）。
- **icon/** — App 图标「网格成勾」
  - `icon.ico` — **Windows 多尺寸图标**（16/24/32/48/256 五尺寸内嵌），可直接用于打包
  - `icon.svg` — 矢量源文件（可无损再导出任意尺寸）
  - `icon-512 / 256 / 48 / 32 / 24 / 16.png` — 各尺寸位图
- **prototype/** — 可点击的交互原型「活的规格」
  - `考勤复核工作台.html` — 双击在浏览器打开，可体验三阶段全流程、所有状态与动效；数值有歧义时以此为准
  - 其余 `.jsx / .css / .js` 为原型源码（React + Babel，仅供参考）

## 阅读顺序

1. 打开 **设计交付说明.html** 通读规范（左侧导航分 12 节）。
2. 打开 **prototype/考勤复核工作台.html** 实际操作：
   - 阶段①右上角「演示状态」切换 空态 / 解析中 / 校验通过 / 有警告 / 缺表
   - 点「开始复核」进入三栏工作台：点击待定单元改判 → 观察即时重算
   - 工具栏可开关 Tweaks（界面气质 / 强调色 / 深色主题）
3. 实现时对照第 10 节（数据与计算规则）和第 11 节（Qt 落地备注）。

## 设计要点一句话

界面强调色（石板灰）与**考勤语义色**严格分离；每个考勤结论 = 颜色 + 专属角标图标 + 边框/纹理三重编码，色弱友好，化解了「红色多义、蓝色多义」的冲突。
