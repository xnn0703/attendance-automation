# 南京六部考勤自动化 — 产品 / 交互 / UI 设计简报

> **给谁看**：UI/UX 设计师。**用来做什么**：理解项目，照本简报做交互原型与视觉方向。
> **不需要懂代码**。读完你应能掌握：业务是什么、谁在用、现在的界面差在哪、我建议的"最优交互逻辑"、以及 UI 的大致方向。
> 文末有"需要你产出的原型清单"和"约束清单"。

---

## 1. 一句话项目

把每月考勤，从「人对着三张 Excel 表手工算大半天、还容易错」，变成「**工具自动算能算的，只在少数真正需要人判断的地方问人**，最后产出一张配色清晰的《考勤》表」的**桌面小工具**（Windows）。

现状：已有一个能跑通的 Python + Qt 桌面程序 + 同源的命令行引擎；规则已定稿，并用 2026-04 真实数据验证（工具比人工还准，抓出了手工表里的 5 处错误）。**现在要把"能用"的界面，升级成"好用、好看、不易错"。**

---

## 2. 用户与使用场景

- **用户**：HR / 行政（1 人操作），**每月跑一次**，在自己的 Windows 电脑上。
- **不是程序员**：要的是清楚、可控、能随时纠错、对结果有信心。
- **对"颜色"很敏感**：在这张表里，**颜色就是结论**（红=迟到/缺勤、绿=缺卡……）。颜色对不对，直接决定表对不对。
- **目标**：又快又准地产出当月《考勤》表，交给上级/财务。

---

## 3. 数据流：3 张输入表 → 1 张成品

| 角色 | 文件 | 内容（设计师只需知道大概） |
|------|------|------|
| 输入① | 打卡原表（钉钉导出） | 每人每天的打卡时间（一天可多次；带"外勤"标记） |
| 输入② | 员工花名册 | 姓名、工号、职位、用工类型、入/离职日期 |
| 输入③ | 调班表 | 当月"上班/休息"日历 + 个人调班名单 |
| 中间件 | 《初表》 | 整理后的打卡表（删无关人、补工号职位、排序） |
| **成品** | **《考勤》** | **按天的网格：每人两行（打卡时间 + 加班时长），单元格用颜色标注考勤结论** |

**成品长这样（关键）**：横向是 1～31 号（每列一天，含周几/班休），纵向是每个人。每人占**两行**：上行是当天打卡时间、下行是当天加班小时数。**单元格的底色/字色表达结论**。

---

## 4. 业务规则要点（解释"为什么有这些颜色和判断"）

工具自动完成：识别工作日/节假日（来自调班表，含"个人调班=把某些天的班↔休对调"）、算加班、判迟到/早退、判外勤、判全勤。**只有两类事必须人来定**（见下）。

### 颜色语义（**这是 UI 的核心，请重点设计**）

| 结论 | 现在的标记方式 | 现用色值 |
|------|----------------|----------|
| 迟到（9:00–9:30 到） | 当天整格**字体**红 | `FF0000` |
| 迟到较重（晚于 9:30；个别人 9:05） | **底色**红 | `FF0000` |
| 早退（早于 18:30 走） | **底色**红 | `FF0000` |
| 缺卡（当天只打了 1 次卡） | 底色绿 | `92D050` |
| 公出（外出公干） | 底色浅蓝 | `DDEBF7` |
| 未出勤（没来） | 底色红 | `FF0000` |
| 外勤日（当天有外勤打卡） | 底色浅蓝 | `DDEBF7` |
| 全勤（整月无异常） | 姓名格底色浅紫 | `D09ECE` |
| 调班上班日（人工备注用） | 底色橙 | `FFFFC000` |

> ⚠️ **现有颜色语义有冲突，需要你帮忙梳理**：红色同时表示「迟到较重 / 早退 / 未出勤」3 种；浅蓝同时表示「公出 / 外勤日」2 种。靠颜色无法区分具体是哪种，得靠人记忆或点开看。**这是 UI 提升的重点之一**（见第 7、8 节）。

### 两个"必须人来定"的交互点（**半自动协作的灵魂**）

1. **离职且当月打卡 < 7 天的人**：留还是删？（逐个问）
2. **某天打卡 < 2 次**：到底是 **缺卡 / 公出 / 未出勤**？（逐个归类；工具给建议值）

其余全部自动。**好的 UI 应该让这两件事变得轻松、有上下文、可批量**，而不是让人在表格里大海捞针。

## 5. 现状界面（要被改进的对象）

当前桌面程序是一个**单窗口 + 5 个标签页的向导**：

```
┌───────────────────────────────────────────────┐
│ [保存决策] [载入决策]   （顶部一行）             │
├───────────────────────────────────────────────┤
│ 标签页： ①选目录 ②整理 ③归类 ④配置 ⑤生成        │
│ ┌───────────────────────────────────────────┐ │
│ │ (当前标签页的内容：表格 / 勾选框 / 下拉 / 按钮)│ │
│ └───────────────────────────────────────────┘ │
├───────────────────────────────────────────────┤
│ 日志：一行行文字输出……                          │
└───────────────────────────────────────────────┘
```

- ①选目录：选文件夹 → 一张"识别到的文件"小表 + 红字校验提示 + 「确认」按钮。
- ②整理：点「运行整理」→ 看纳入名单表 + 离职<7天勾选 + 删除名单 → 「应用」。
- ③归类：点「列出待定项」→ 一张表，每行一个下拉（缺卡/公出/未出勤）→ 「保存」。
- ④配置：3 个输入框（不计加班名单 / 严格迟到阈值 / 只红字名单）。
- ⑤生成：点「生成」→ 汇总表 + 一大块彩色预览网格 + 「打开/重算」。

## 6. 现状交互的不足（诚实评估）

1. **标签页割裂、流程感弱**：5 个 tab 要手动点；步骤之间的依赖（必须先①再②…）只靠"灰掉后面的 tab"，不够引导。
2. **处处要手动按钮触发**（"运行整理""列待定项""生成"）：工具明明能自动跑，却让用户一步步点。
3. **彩色网格密集难读**：50+ 行 × 37 列的小格子，没有冻结表头/姓名列、没有图例、不能搜人、不能跳到异常，看着累。
4. **归类是一张枯燥的下拉表**：几十条逐个拉，没有上下文（看不到当天打卡）、没有"连续空白可能是公出"的提示、没有"按人分组/批量"。
5. **没有总览和进度**：不知道总共多少人、还剩几处没归类、有多少异常、整体进度到哪。
6. **反馈偏工程师风**：底部日志框、报错弹窗；缺少行内提示、颜色含义的悬浮说明、撤销。
7. **两个关键决策点被"埋"在标签页里**，没有被当作主角呈现。
8. **没有结果仪表盘**：加班合计、全勤名单、异常清单等没有好看的汇总视图。
9. **视觉是 Qt 默认样式**：朴素、缺乏统一视觉语言；颜色语义又有冲突（见第 4 节）。

## 7. 我建议的"最优"交互逻辑

### 设计原则（5 条）

1. **自动优先**：用户只管"选目录"，工具立刻把能算的都算完，把结果摆出来。不要让用户一步步点"运行 X"。
2. **人只做判断**：把界面焦点放在那两个"必须人定"的点上，且给足上下文 + 建议 + 批量。
3. **结论可视、可改、即时**：彩色表是主角；任何改判/改配置 → **秒级重算并刷新**。
4. **防错优先**：月份不一致、缺表、数据异常，要在最显眼处温和拦截并指路（这次真实数据就因为月份对不上出过错）。
5. **可控可回退**：每步可回看、可改、可撤销；导出前能完整预览。

### 把"5 个标签页"重构成"3 个阶段 + 1 个工作台"

```
   ① 准备                ② 复核与决策（核心工作台）            ③ 完成
 选目录 → 自动就绪   →   左：待办清单   中：彩色考勤表   右：明细/检查器   →   汇总仪表盘 + 导出
```

#### 阶段 ① 准备（轻、快）
- 选/拖入数据文件夹 → 工具**自动识别三表、校验、解析、并把能算的全算完**。
- 给一张「**就绪卡片**」：本月是几月、纳入多少人、校验结果（✓/⚠ 如"原表是3月、调班表是4月"）、**待办 N 项**（离职取舍 x 个 + 待归类 y 个）。
- 一个明显的「开始复核 →」。

#### 阶段 ② 复核与决策（**主屏，一个三栏工作台**）
这是整个产品的核心。把"整理/归类/配置"融进**一个**实时联动的界面：

- **左栏 = 待办清单（To-Do）**：把所有"需要人定"的事做成一个可勾掉的清单：
  - 「离职取舍」：列出离职且打卡<7天的人（带打卡天数），勾选保留谁。
  - 「逐条归类」：每个"打卡<2次"的案例一行，显示 姓名·日期·当天打卡内容·建议值，**一键采纳建议** / 下拉改判；**连续空白、外勤单卡等"疑似公出"自动高亮**并给提示。
  - 顶部进度："待办 12 项 / 已处理 7"。
- **中栏 = 彩色考勤表（核心组件，见下）**：实时反映当前所有判断与配色。
- **右栏 = 明细 / 检查器**：选中某人或某格时，显示该格的**原始打卡、判定原因（为什么红/绿）、加班怎么算出来的**；以及该人的小结（加班合计、是否全勤）。
- **联动**：点左栏某条 → 中栏网格**高亮定位**到对应格；在网格上**直接点格改判** → 左栏清单与右栏明细同步、**整表秒级重算**。
- **配置**默认收进「高级设置」抽屉（不计加班名单、个别人迟到阈值），多数月份不用动。

#### 阶段 ③ 完成
- **汇总仪表盘**：本月概览（人数、加班合计、全勤名单、迟到/缺卡/未出勤 计数、异常清单）。
- **完整预览** + 「导出《考勤》」（生成 Excel）+「打开文件」。
- 「保存本月决策 / 下月载入」（复用上月的归类与配置，减少重复）。

### 中栏核心组件：彩色考勤表（请重点设计）
- **冻结**：左侧姓名列 + 顶部日期表头始终可见；日期表头标**周几 + 班/休**（班休来自调班表）。
- **日历底纹**：休息日/节假日列用浅底纹，让人一眼看出"这天本就该休"。
- **每人两行**：打卡行 + 加班行（或用更聪明的展开/合并方式，由你设计）。
- **待办格高亮**：需要人归类的格子加角标/描边，可点；点击就地选 缺卡/公出/未出勤。
- **悬浮即解释**：鼠标悬停一格 → 浮层显示当天打卡明细 + 为什么是这个颜色 + 加班算式。
- **找人/筛选**：搜索姓名、只看有异常的人、跳到下一个异常。
- **常驻图例**：颜色 → 含义对照，始终可见。

### 即时反馈 / 状态
- 任何改动（改判、改配置、改保留名单）→ 网格与汇总**即时刷新**（无需"重算"按钮）。
- 顶部一条状态：当前阶段、待办计数、保存/导出状态；操作有 Toast 轻提示；支持撤销。

### 边界与异常（要设计"出错时长什么样"）
- 月份不一致 / 缺某张表 / 文件格式异常 / 模板不符 → 在就绪卡片或顶部**醒目但温和**地提示，并给"怎么办"（如"请换成 4 月的原表"）。
- 空态（还没选目录）、加载态（解析中）、完成态都要有相应画面。

## 8. UI 的大致方向（视觉与布局）

### 整体气质
**"专业表格" × "被引导的向导"**：冷静、克制、信息密度高但读得清。参考感觉：现代的财务/数据审阅类桌面工具（清爽中性色 + 一处强调色 + 语义色彩用于数据）。

### 布局骨架（阶段②工作台）
```
┌──────────────────────────────────────────────────────────────┐
│ 顶部状态条： 数据目录 ｜ 本月X月·N人 ｜ 待办12/已处理7 ｜ [导出]   │
├──────────┬───────────────────────────────────────┬───────────┤
│ 待办清单  │   彩色考勤表（冻结姓名列+日期表头）       │ 明细/检查器 │
│ □离职取舍 │   ┌─────────────────────────────────┐  │ 选中格的：  │
│ ☑归类#1   │   │ 姓名│1 2 3 六 …（周几/班休/底纹） │  │ ·原始打卡   │
│ ☑归类#2   │   │ 张三│■ ■ ▣ …                      │  │ ·判定原因   │
│ □归类#3…  │   │ 李四│…                            │  │ ·加班算式   │
│ [高级设置]│   └─────────────────────────────────┘  │ 人小结      │
│           │   图例： ■红=迟到 ■绿=缺卡 …            │            │
└──────────┴───────────────────────────────────────┴───────────┘
```

### 颜色系统（**关键，请重新梳理**）
- **把"数据语义色"和"界面装饰色"分开**：界面用中性灰 + 1 个品牌强调色；考勤结论才用语义色。
- **化解现有冲突**（红/蓝各有多义）：建议用「**颜色 + 图标/纹理 + 悬浮文字**」共同表意，而不只靠底色。例如：
  - 迟到/早退/未出勤都偏红 → 可用不同图标或深浅/边框区分（迟到=轻、早退=右下角、未出勤=实心红+"缺"字等）。
  - 公出 vs 外勤日都偏蓝 → 一个用底色、一个用左边竖条/斜纹。
  - 调班上班日（橙）目前是人工备注，可考虑工具自动呈现"今天是调来的班"。
- **无障碍**：考虑色弱友好（不仅靠红绿区分）。
- 提供浅色为主、（可选）深色主题。

### 组件清单（给设计师的"零件表"）
就绪卡片 / 待办清单项（含建议徽标）/ 彩色网格（冻结、底纹、角标、悬浮卡）/ 图例 / 检查器面板 / 汇总仪表盘卡片 / 顶部状态条 / 导出对话 / 空·载入·错误态 / Toast 通知 / 高级设置抽屉。

### 技术约束（影响可实现性，请知悉）
- **桌面应用，Windows，技术栈是 Python + Qt（PySide6）**。原型视觉应是**Qt 可实现**的范畴：可深度自定义样式（类似 QSS 主题）、可自绘表格单元格；但**不要依赖纯网页式的复杂动效**。表格是 Qt 的 `QTableView` + 自绘委托能力——冻结表头、单元格自绘、悬浮卡都可做。
- 单窗口、单用户、本地运行、**无登录/无云/无多人协作**。
- 内容全中文（含中文字体选择）。

## 9. 需要你（设计师）产出的原型清单

1. **就绪页**（阶段①）：空态 / 已选目录·校验通过 / 校验有警告（如月份不一致）三种。
2. **复核工作台**（阶段②）：三栏布局；含"待办清单""彩色网格""检查器"；以及"全部处理完"的状态。
3. **彩色网格组件细节**：冻结、日历底纹、待办格高亮、悬浮解释卡、图例、搜索/筛选。
4. **单格改判**交互：在网格里点一个缺卡格 → 选 缺卡/公出/未出勤 → 即时变色。
5. **汇总仪表盘 + 导出**（阶段③）。
6. **异常/空/加载态**：缺表、月份不一致、解析中、无数据。
7. **颜色系统规范**：语义色 + 图标/纹理 + 图例的一套完整方案（化解现有多义冲突）。

## 10. 约束 / 边界（一页速览）
- 平台：Windows 桌面；技术：PySide6/Qt（自定义样式可，纯 web 动效不可）。
- 用户：1 名 HR/行政，月度使用，非技术。
- 数据：本地、含个人信息（不上云、无登录）。
- 半自动：工具自动算；人只做「离职取舍」「<2次打卡归类」两类判断 + 复核。
- 产出：一张配色《考勤》Excel；界面内的彩色表应与之一致。
- 颜色即结论：颜色系统是本设计的重中之重。
