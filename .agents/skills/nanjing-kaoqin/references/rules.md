# 南京六部考勤规则（权威口径）

与用户逐条确认的固定规则，与月份无关。`scripts/kq_run.ps1` 即按此实现。时间以「当天 0 点起的分钟数」计。

## 阶段一 · 表格整理 → 《初表》

1. 删除原表中**部门 ≠ 南京六部**的人员（如「总部综合部」）。
2. 按**姓名**从《花名册》匹配 **工号、职位** 填入；再按**工号升序**排序。
   ⚠️ 排序时**整行（含每日打卡）随姓名一起移动，不能错位**。
3. **重名**：参考花名册 `用工类型`，取**在职**（转正/试用）那条（入离职区间覆盖当月）。
4. **离职且打卡 < 7 天**：整理姓名**询问用户**保留谁；未保留者删除其数据。保留的离职人员**只算到离职日**。
5. 另存为 **《初表》**。

## 阶段二 · 考勤核算（在《初表》上完成 → 《考勤》）

### A. 工作日 / 节假日判定
- 基准日历=《调班表》左半区网格（`N班`=上班 / `N休`=休息）。
- **个人调班（右半区名单）**：每人「原日期/班次」+「现调日期/班次」两列涉及的**当月日期，一律按基准日历翻转 班↔休**
  （两列谁=上班/休息**不固定**，由基准决定：基准班→改休、基准休→改班）。脚本自动解析，**不要硬编码日历、也不要写死"现调=上班"**。
- ⚠️ 原表与《调班表》月份须一致（以原表 A1 为准）。无《调班表》→ 询问是否补充；不补则按正常法定日历。

### B. 加班（OT）
- **排除人员**：见 `kq_config.txt` 的 `OT_EXCLUDE`（默认 陆钊颖、曹兆阳）——不计加班。
- **有效性**：当天 **≥2 次打卡**才计 OT。
- **取整**：时间点按 **0.5 小时向内取整**（起点向上、终点向下）。例 `9:15–10:15` → `9:30–10:00` = 0.5h。
- **下限**：**9:00 前的打卡不计**（OT 起点 ≥ 9:00）。
- **工作日 OT** = 末次打卡(向下取整) − `18:30`；**≥2h（含）则减 0.5h**；<0 记 0。
- **节假日 OT** = 末次(向下取整) − 首次(向上取整且 ≥9:00)；**跨 `12:00–13:30` 则扣 1.5h**；**>8h（不含）则减 0.5h**。
- **外勤**：**工作日外勤不计加班**；**节假日外勤正常计，但末点封顶 21:00**（超 21:00 不计）。
- **取整顺序**：先取整时间点 → 算时长 → 再扣减；**跨午夜按 +24h 连续计算**。
- **写法**：每人打卡行**下方加一行**，把当天 OT 小时数填到对应日期格——**只填数字、无单位**，无 OT 留空。

### C. 迟到 / 早退（仅工作日；全员含外勤，无一例外）
- **迟到**：首次打卡 **>9:00** → 当日整格**字体红 `FF0000`**（当天所有时间点都红）。
- **迟到加重**：首次 **>9:30** → 字体不变、**底色红 `FF0000`**。
- **刘莹特例**（`STRICT_LATE`）：**>9:05** 即按加重处理（底色红）。
- **曹兆阳特例**（`LATE_FONT_ONLY`）：迟到**一律仅字体红**（>9:00 即红字），**不适用 >9:30 底色红**（早退等不受影响）。
- **新入职员工·入职当天**：迟到同样**仅字体红、不底色红**（仅当天；脚本据花名册入职日自动判定）。
- **早退**：末次打卡 **<18:30** → 字体不变、**底色红 `FF0000`**。**外勤打卡不适用早退**，其余全员适用。

### D. 缺卡 / 公出 / 未出勤
- **2 次打卡 = 有效出勤**（工作日、节假日加班同此）。
- **工作日打卡 < 2 次** → **询问用户**归类：缺卡=底色**绿 `92D050`** / 公出=**浅蓝 `DDEBF7`** / 未出勤=**红 `FF0000`**。
  （`kq_classify.txt`：B=公出 / G=缺卡 / R=未出勤。建议默认 0 次=未出勤、1 次=缺卡。）
- **节假日加班 < 2 次** → **不标记、当日不计 OT**。
- 缺卡与迟到/早退**可并存**；底色冲突 → **字体红 `FF0000` + 底色绿 `92D050`**。

### E. 外勤
- **外勤日认定**：当天**只要有任一「外勤」打卡**，整天即按外勤日（混合打卡也算）。
- 工作日/节假日判定**与常人相同**（法定日历 + 调班表）。
- **外勤日整格底色浅蓝 `DDEBF7`**；该日若打卡<2次出现缺卡/未出勤/公出，按 D 条处理（覆盖外勤蓝）。

### F. 背景色优先级（高 → 低）
1. **工作日 <2 次归类**（三选一，最高）：未出勤红 / 缺卡绿 / 公出蓝。
2. **外勤日**：浅蓝。
3. **迟到>9:30 / 早退**：红（最低）——**外勤日时被外勤蓝覆盖、不生效**。
- 缺卡+迟到/早退 → 字体红+底色绿。
- 迟到 9:00–9:30 是**字体红**（不占底色），可与任何底色并存。

### G. 在职区间
- 仅对**在职区间内**（入职日 ≤ 当日 ≤ 离职日；无离职日则到月末）做判定与着色。
- **入职前 / 离职后的工作日空格**：**留空、不标记**，不计迟到/缺卡/未出勤，**也不影响全勤**。

### H. 全勤
- 在职区间内的工作日**无 迟到/早退/缺卡/未出勤** → **全勤**，**姓名格底色浅紫 `D09ECE`**。（公出、外勤不破坏全勤。）

## 应出勤天数（如需）
- 通则：**应出勤 = 当月实际出勤工作日数 + 当月法定节假日天数**。
- 新入职人员应出勤**与其他人相同、不按入职日折算**。
- 注：当前《考勤》成品未单列「应出勤」列；如需可另加。

## 人工交互点
1. 阶段一：离职且打卡<7天 → 问保留谁。
2. 阶段二 D：工作日打卡<2次 → 逐条问 缺卡/公出/未出勤。
3. 阶段二 A：缺《调班表》 → 问是否补充。
4. 任何规则冲突/特殊 → 问用户。
