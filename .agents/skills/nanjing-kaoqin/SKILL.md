# 南京六部月度考勤（Nanjing-Liu-Bu Monthly Attendance）

把一个月的「钉钉打卡导出 + 花名册 + 调班表」核算成一张配色考勤表。**整套规则是固定的、与月份无关**
（详见 `references/rules.md`），每月变化的只有：三张输入表、少量人工归类、以及偶尔的人员策略配置。

你的职责是**驱动这条三阶段流水线，并在脚本停下来要人工判断时与用户交互**。脚本做确定性计算，
你做编排 + 把需要人判断的清单摆给用户。

## 输入与产出（同一工作目录内）

- 输入：`*原表*.xlsx`（钉钉打卡导出）、`*花名册*.xlsx`、`*调班表*.xlsx`（按文件名子串识别，大小写/前后缀不限）。
- 中间件：`*（初表）*.xlsx`（阶段一产出）。
- 成品：`*（考勤）*.xlsx`（阶段二产出）。
- 临时配置（脚本读，你来写）：`kq_keep.txt`、`kq_classify.txt`、`kq_config.txt`（格式见末尾）。

## 环境要点（必读，否则会踩坑）

- **本机无可用 Python**（openpyxl/pandas 不可用）。一切靠 `scripts/kq_run.ps1`（纯 PowerShell，把 xlsx 当 zip 解析/改写）。
- **调用脚本必须用 PowerShell 工具的 `&` 调用算子，不要用 `powershell.exe -File`**——脚本路径/数据目录含中文「考勤」，
  嵌套 `powershell.exe` 传参会被中文截断。正确写法：
  ```powershell
  & "<此skill目录>\scripts\kq_run.ps1" -Stage prep -Dir "D:\考勤"
  ```
- 脚本是纯 ASCII（中文用 code points 构造）。**改脚本时切记 PowerShell 变量名大小写不敏感**（`$DAYS` 与 `$days`
  是同一变量）、**函数名别用内置别名**（`gc`/`cp`/`ls`…）——这两个坑本项目已反复踩到。

## 流程（四步）

设 `$S = "<此skill目录>\scripts\kq_run.ps1"`，`$D = "<本月数据目录>"`（如 `D:\考勤`）。

### 第 1 步 · 整理 → 《初表》

```powershell
& $S -Stage prep -Dir $D
```
脚本会：删非「南京六部」部门的人；按姓名从花名册补 工号/职位；工号升序；删「离职且打卡<7天」者；
然后打印 KEPT 名单、DROPPED 名单、以及 **离职<7天候选**。

**人工交互（离职<7天）**：若打印出「LIZHI-<7 CANDIDATES」，把这些姓名+打卡天数**摆给用户问保留谁**。
按用户答复，把要**保留**的工号写进 `$D\kq_keep.txt`（每行/逗号分隔），再 `-Stage prep` 重跑一次。
（被保留的离职人员只算到离职日——脚本会自动处理在岗区间。）未保留的即从初表删除。

### 第 2 步 · 列异常 → 人工归类

```powershell
& $S -Stage worklist -Dir $D
```
脚本列出**所有「工作日打卡<2次」**的格子（含 工号|日、打卡次数、是否外勤 WQ、建议值）。建议默认：
**0 次=R(未出勤)、1 次=G(缺卡)**。

**人工交互（核心）**：把这份清单摆给用户逐条确认。重点提示用户：
- 连续多天空白、或外勤当天只打 1 次卡，**常常是「公出(B)」而非默认值**；
- 用户可说"全部按建议"，再列出要改的（如"#9 改 B"）。

把最终归类写进 `$D\kq_classify.txt`，每行 `工号|日=B|G|R`（B=公出/蓝, G=缺卡/绿, R=未出勤/红）。
**清单上每一条都要有归类**（默认值也要落进文件，便于复跑可复现）。

### 第 3 步 · 确认人员策略配置（通常沿用上月）

`$D\kq_config.txt` 控制每月可能变的人员策略。**文件不存在时脚本用内置默认**（即南京六部当前口径）：
- `OT_EXCLUDE`（不计加班）：陆钊颖 `Y17074`、曹兆阳 `Y28001`
- `STRICT_LATE`（迟到底色红的更严阈值）：刘莹 `Y28006:0905`（其余人默认 09:30）
- `LATE_FONT_ONLY`（迟到只红字、不红底）：曹兆阳 `Y28001`（新入职员工入职当天自动适用，无需配置）

若这些人员/口径本月有变，写一个 `kq_config.txt`（格式见末尾）覆盖默认。否则可跳过本步。

### 第 4 步 · 核算 → 《考勤》

```powershell
& $S -Stage build -Dir $D
```
脚本按 `references/rules.md` 全套规则核算（加班/迟到/早退/缺卡-公出-未出勤/外勤/全勤），改样式上色、
每人下方插「加班时长」行，产出《考勤》，并打印每人 加班合计 + 是否全勤 的汇总。

**核对**：粗看汇总（加班量级、全勤名单）是否合理；可抽查个别单元格 `s` 属性是否符合预期色值。
然后把成品交给用户复核。用户要调规则口径→改 `scripts/kq_run.ps1` 或 `kq_config.txt`/`kq_classify.txt` 重跑第 4 步即可。

## 配置文件格式

`kq_keep.txt`（要保留的离职人员工号；可空）：
```
Y28068, Y28099
```

`kq_classify.txt`（工作日<2次打卡的归类；每行一条）：
```
# 工号|日 = B(公出/蓝) | G(缺卡/绿) | R(未出勤/红)
Y28001|9=B
Y28020|14=R
Y28005|22=G
```

`kq_config.txt`（人员策略；可选，缺省=南京六部当前口径）：
```
OT_EXCLUDE=Y17074,Y28001
STRICT_LATE=Y28006:0905
LATE_FONT_ONLY=Y28001
```

## 颜色方案（色值，详见 references/rules.md 的「F. 背景色优先级」）

| 情形 | 方式 | 色值 |
|------|------|------|
| 迟到 9:00<首卡≤9:30 | 字体红 | FF0000 |
| 迟到>9:30（刘莹>9:05；外勤日除外） | 底色红 | FF0000 |
| 早退（末卡<18:30，非外勤） | 底色红 | FF0000 |
| 缺卡 | 底色绿 | 92D050 |
| 公出 | 底色浅蓝 | DDEBF7 |
| 未出勤 | 底色红 | FF0000 |
| 外勤日 | 底色浅蓝 | DDEBF7 |
| 全勤（姓名格） | 底色浅紫 | D09ECE |
| 缺卡+迟到/早退 | 字体红+底色绿 | FF0000+92D050 |

## 规则

**完整、权威的考勤规则在 [`references/rules.md`](references/rules.md)**——表格整理、加班核算（取整/午休/外勤/封顶）、
迟到早退（含刘莹/曹兆阳/新人特例）、缺卡公出未出勤、外勤、全勤、应出勤天数、背景色优先级。改任何计算口径前先读它。

## 排错

- 报「positional parameter ... 'A'」或参数被截断 → 你用了 `powershell.exe -File` 且路径含中文；改用 `&` 调用。
- 报「无原表/初表/调班表」→ 文件名不含对应子串（原表/初表/调班/花名册），或没在 `-Dir` 指定目录。
- `build` 找不到《初表》→ 先跑 `prep`。
- 颜色/加班明显不对 → 多半是 `kq_classify.txt` 或 `kq_config.txt` 没更新；其次才查 `scripts/kq_run.ps1`。
- 改脚本后结果异常 → 优先排查变量名大小写撞车、函数名撞内置别名（见「环境要点」）。
